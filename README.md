# CrossCopy

在同一局域网的 macOS 和 Windows 电脑之间共享鼠标，并传递文本、文件、文件夹或压缩包。默认使用 `Ctrl+Shift+M` 开关鼠标共享，使用 `Ctrl+Shift+C` 跨设备复制，使用 `Ctrl+Shift+V` 跨设备粘贴。

## 已实现

- UDP 组播、全局广播和网卡定向广播自动发现设备，不需要填写 IP 或端口
- 首次使用 6 位验证码配对，验证码 2 分钟后失效
- 配对后保存 256 位随机密钥
- 文本、任意文件、多个文件、文件夹和压缩包传输
- AES-256-GCM 加密控制消息和每一个文件数据块
- 专用全局快捷键触发，普通复制粘贴不触发网络
- 设置页可修改跨设备复制和粘贴快捷键
- 鼠标页面可拖动设置另一台电脑位于本机的上、下、左或右
- 鼠标从对应屏幕边缘穿越，并按比例保持横向或纵向位置
- 支持鼠标移动、左键、右键、滚轮滚动和滚轮按下
- 任一电脑关闭鼠标共享时，配对端同步关闭
- 目标电脑的物理鼠标输入优先，可立即中止远程鼠标会话
- 鼠标移动使用配对密钥加密的 UDP 数据包，并显示 RTT/2 延时估算
- 未开启鼠标共享时不启动鼠标监听，也不产生鼠标网络流量
- 接收内容保存在 CrossCopy 待粘贴区，不长期覆盖系统剪贴板
- 1 MiB 固定缓冲流式传输，大文件不会整体载入内存
- 大文件使用独立进度窗口，关闭主窗口后仍可查看
- 文件下载到系统“下载/CrossCopy”目录
- 空闲后自动降频，快捷键触发时立即唤醒设备发现
- 托盘后台运行、暂停同步、开机启动、浅色和深色模式
- 本地滚动日志和一键导出诊断报告

macOS 首次使用时，请在 CrossCopy 设置页点击“打开系统设置”，授予
CrossCopy“辅助功能”权限，用于跨设备快捷键和鼠标控制。
CrossCopy 不会在每次按快捷键时重复弹出系统授权窗口。关闭主窗口不会
退出 CrossCopy，可从菜单栏托盘图标重新打开。

## 导出诊断日志

在主界面底部点击“导出诊断日志”。CrossCopy 会在系统下载目录的
`CrossCopy` 文件夹中生成 `CrossCopy-diagnostics-*.txt`，并自动在
Finder 或资源管理器中选中该文件。

诊断日志不会记录验证码、设备密钥、剪贴板文本内容或完整文件路径。
单个日志文件最大 2 MB，并最多保留 3 份历史日志。

## 开发

要求 Node.js 22、Rust stable，以及对应平台的 Tauri 系统依赖。

```bash
npm install
npm run dev
```

运行检查：

```bash
npm run typecheck
npm test
```

## 打包

### 一键同时打包 macOS 和 Windows

将代码推送到 GitHub 后，打开仓库的 **Actions** 页面，选择
**一键打包 Mac 和 Windows**，点击 **Run workflow**。完成后会得到：

- `CrossCopy-macOS-Universal`：同时支持 Intel 和 Apple Silicon 的 DMG
- `CrossCopy-Windows-x64`：Windows NSIS EXE 和 MSI 安装包

如果本机已安装并登录 GitHub CLI，也可以运行：

```bash
npm run release:all
```

该命令会触发云端双平台构建、等待完成，并把安装包下载到
`release/<任务编号>`。

### 单独在当前系统打包

macOS 安装包需要在 macOS 构建：

```bash
npm run build
```

Windows 安装包需要在 Windows 构建，建议使用同一条命令或 CI：

```powershell
npm install
npm run build
```

流水线会给没有开发者证书的 macOS 产物写入固定的本地应用身份，使
辅助功能授权在版本升级后保持稳定；这不等同于 Apple 公证。未使用
Developer ID 或 Windows 代码签名证书的安装包仍会触发 Gatekeeper 或
SmartScreen 提示，公开分发前应配置正式签名证书。

## 网络说明

CrossCopy 只在局域网内工作。企业 Wi-Fi、访客网络或开启 AP 隔离的路由器可能禁止设备互相发现。系统防火墙首次询问时，需要允许 CrossCopy 接收局域网连接。

配对码是短时认证凭据。配对后的内容使用设备密钥加密，但首次配对仍应在可信局域网中完成。

## 性能设计

- 传输文件时不生成 zip、tar 或中间副本
- 已压缩文件不会被重复压缩
- 大文件按 1 MiB 块读取、加密、写入
- 每块只增加 28 字节加密开销
- TCP `NODELAY` 降低短文本延迟
- 鼠标实时事件使用有界队列，界面状态最多每 100 ms 刷新一次
- 鼠标监听按需启动，且只监听鼠标事件，不读取或转发键盘
- Release 构建开启 LTO、符号裁剪和单代码生成单元

实际速度取决于网卡、Wi-Fi 信号、磁盘速度和杀毒软件。千兆有线局域网通常由磁盘或链路带宽决定，而不是 CrossCopy 的缓冲区。
